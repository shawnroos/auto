# Scoped build prompt

Implement exactly the one change described for the target — no more. Stay inside
the stated scope; do not refactor adjacent code or gold-plate.

- Follow the conventions already present in the files you touch.
- Add or extend a test that would fail without your change, and see it pass.
- Leave the change durable (written to disk); do not commit unless asked.

If the scope is ambiguous, make the smallest reasonable interpretation and state
the assumption in your report rather than expanding the work.
