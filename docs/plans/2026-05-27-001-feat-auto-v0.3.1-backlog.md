## Backlog items

### B1. `/auto-status --json` mode (P2 carry-over)

**Origin:** round-2 cli-r2-001, widened in round-3 by G2's `exit_reason`
addition (cli-r3-004). Same finding across two rounds.

**Problem:** `/auto-status` output is prose-only. With v0.3.0 the iteration
section + bound_exit sub-bullet + exit_reason line push the field count past
the threshold where ad-hoc per-consumer parsing is the wrong shape. An agent
inspecting iteration state must parse human-readable lines that aren't
contract-stable.

**Shape of the fix:** add `--json` flag to `lib/auto-status.sh`'s arg parser.
On `--json`, emit a structured envelope (one ledger → one JSON object) with
known keys: `run_id`, `loop_phase`, `exit_predicate_result`, `iteration`
(when present), `exit_reason` (when present), `units` (per-unit summaries).
Prose output stays the default; agents pass `--json` to get a parseable
surface. Add a test asserting the JSON envelope shape is stable.

**Why deferred:** functional impact is "agents work harder," not "feature
broken." No ship-blocker. v0.3.1 is the right home.

### B2. doc-fence auto-discovery upgrade (defense-completion)

**Origin:** advisor framing on H ship review. `tests/unit/doc-fence-ledger-schema.test.sh`
catches **one direction** of the class — "REQUIRED symbol missing from doc."
It does NOT catch the inverse — "code added new public symbol, but `REQUIRED`
list in the test wasn't updated."

**Problem:** the maintenance contract for the lint requires the engineer adding
a new public `lib/ledger.py` symbol to ALSO update the test's `REQUIRED` list.
That's exactly the convention-enforced trap `feedback_deterministic_over_probabilistic_v1`
calls out — if the engineer forgets, the lint stays green and the doc-drift
recurs silently.

**Shape of the fix:** rewrite the lint to auto-discover public symbols:
- For functions: walk `lib/ledger.py` via `ast.parse`, collect `FunctionDef`
  names that don't start with `_` (and aren't aliased re-imports).
- For module constants: collect `Assign` targets at module level whose name
  is `ALL_CAPS` (excluding the `_PRIVATE` convention).
- For each discovered symbol, assert it's mentioned in
  `docs/contracts/ledger-schema.md`. Drop the manually-maintained `REQUIRED`
  list entirely.

**Why deferred:** the round-3 cited instance (API-R3-1: `exit_reason` +
`set_exit_reason` undocumented) IS closed by H's current half-defense. The
auto-discovery upgrade is "close the class" not "fix the instance." v0.3.1
work.

### B3. shape-guard pattern at 6+ sibling sites (class-completion)

**Origin:** round-3 learnings researcher flagged that G2 closed the chokepoint
(`iteration.compute_pending_state`) but the vulnerable pattern
`led.get("iteration") or {}` followed by chained `.get(...)` survives at:

- `lib/emitters.py:108` (`plan_output_to_paired_builders` reading at emit time)
- `lib/emitters.py:213` (`iterate_template` reading at emit time)
- `lib/tick.py:616` (`advance_iteration_loop` reading)
- `lib/tick.py:666` (advance_iteration_loop)
- `lib/tick.py:874` (post-iteration advance)
- `lib/tick.py:1123` (`_iteration_guidance_prefix`)
- `lib/iteration.py:143` (`evaluate_decision`)
- `lib/recipes.py:346` (validator gate_unit lookup)

**Problem:** a torn-ledger write of `iteration` to a non-dict scalar would
crash AttributeError at any of these sites if the chokepoint guard hasn't
already collapsed `iteration_pending` to False (which requires
`recompute_predicate` to have run). Most sites READ the ledger post-mutation,
so they're protected by the chokepoint — but the emit-time sites
(`lib/emitters.py:108,213`) read disk-loaded ledger BEFORE recompute, which
is where the class could still trip.

**Shape of the fix:** introduce a helper in `lib/iteration.py`:
```python
def safe_iteration_block(ledger: dict) -> dict | None:
    """Return ledger.iteration if it's a well-formed dict, else None."""
    block = ledger.get("iteration")
    if not isinstance(block, dict):
        return None
    return block
```
Then replace all 6+ sites with `block = iteration.safe_iteration_block(led)`.
Add an AST lint that forbids `led.get("iteration") or {}` anywhere in `lib/`
(same shape as the existing `iteration-ast-lint.test.sh` for the "decision"
literal).

**Why deferred:** the cited round-2 instance (ADV-R2-1: cascade in
compute_pending_state) IS closed by G2. The 6+ sibling sites require a
specific torn-ledger AND specific gate state to trip — P3-equivalent in
practice. But the class-completion is real v0.3.1 work.
