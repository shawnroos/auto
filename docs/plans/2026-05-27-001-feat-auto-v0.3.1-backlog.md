## Backlog items

### B1. `/auto-status --json` mode (P2 carry-over)

**Origin:** round-2 cli-r2-001, widened in round-3 by G2's `exit_reason`
addition (cli-r3-004). Same finding across two rounds.

**Problem:** `/auto-status` output is prose-only. With v0.3.0 the iteration
section + bound_exit sub-bullet + exit_reason line push the field count past
the threshold where ad-hoc per-consumer parsing is the wrong shape. An agent
inspecting iteration state must parse human-readable lines that aren't
contract-stable.

**Shape of the fix:** add `--json` flag to `lib/auto-status.sh`'s arg parser.
On `--json`, emit a structured envelope (one ledger → one JSON object) with
known keys: `run_id`, `loop_phase`, `exit_predicate_result`, `iteration`
(when present), `exit_reason` (when present), `units` (per-unit summaries).
Prose output stays the default; agents pass `--json` to get a parseable
surface. Add a test asserting the JSON envelope shape is stable.

**Why deferred:** functional impact is "agents work harder," not "feature
broken." No ship-blocker. v0.3.1 is the right home.

### B2. doc-fence auto-discovery upgrade (defense-completion)

**Origin:** advisor framing on H ship review. `tests/unit/doc-fence-ledger-schema.test.sh`
catches **one direction** of the class — "REQUIRED symbol missing from doc."
It does NOT catch the inverse — "code added new public symbol, but `REQUIRED`
list in the test wasn't updated."

**Problem:** the maintenance contract for the lint requires the engineer adding
a new public `lib/ledger.py` symbol to ALSO update the test's `REQUIRED` list.
That's exactly the convention-enforced trap `feedback_deterministic_over_probabilistic_v1`
calls out — if the engineer forgets, the lint stays green and the doc-drift
recurs silently.

**Shape of the fix:** rewrite the lint to auto-discover public symbols:
- For functions: walk `lib/ledger.py` via `ast.parse`, collect `FunctionDef`
  names that don't start with `_` (and aren't aliased re-imports).
- For module constants: collect `Assign` targets at module level whose name
  is `ALL_CAPS` (excluding the `_PRIVATE` convention).
- For each discovered symbol, assert it's mentioned in
  `docs/contracts/ledger-schema.md`. Drop the manually-maintained `REQUIRED`
  list entirely.

**Why deferred:** the round-3 cited instance (API-R3-1: `exit_reason` +
`set_exit_reason` undocumented) IS closed by H's current half-defense. The
auto-discovery upgrade is "close the class" not "fix the instance." v0.3.1
work.

### B3. shape-guard pattern at 6+ sibling sites (class-completion)

**Origin:** round-3 learnings researcher flagged that G2 closed the chokepoint
(`iteration.compute_pending_state`) but the vulnerable pattern
`led.get("iteration") or {}` followed by chained `.get(...)` survives at:

- `lib/emitters.py:108` (`plan_output_to_paired_builders` reading at emit time)
- `lib/emitters.py:213` (`iterate_template` reading at emit time)
- `lib/tick.py:616` (`advance_iteration_loop` reading)
- `lib/tick.py:666` (advance_iteration_loop)
- `lib/tick.py:874` (post-iteration advance)
- `lib/tick.py:1123` (`_iteration_guidance_prefix`)
- `lib/iteration.py:143` (`evaluate_decision`)
- `lib/recipes.py:346` (validator gate_unit lookup)

**Problem:** a torn-ledger write of `iteration` to a non-dict scalar would
crash AttributeError at any of these sites if the chokepoint guard hasn't
already collapsed `iteration_pending` to False (which requires
`recompute_predicate` to have run). Most sites READ the ledger post-mutation,
so they're protected by the chokepoint — but the emit-time sites
(`lib/emitters.py:108,213`) read disk-loaded ledger BEFORE recompute, which
is where the class could still trip.

**Shape of the fix:** introduce a helper in `lib/iteration.py`:
```python
def safe_iteration_block(ledger: dict) -> dict | None:
    """Return ledger.iteration if it's a well-formed dict, else None."""
    block = ledger.get("iteration")
    if not isinstance(block, dict):
        return None
    return block
```
Then replace all 6+ sites with `block = iteration.safe_iteration_block(led)`.
Add an AST lint that forbids `led.get("iteration") or {}` anywhere in `lib/`
(same shape as the existing `iteration-ast-lint.test.sh` for the "decision"
literal).

**Why deferred:** the cited round-2 instance (ADV-R2-1: cascade in
compute_pending_state) IS closed by G2. The 6+ sibling sites require a
specific torn-ledger AND specific gate state to trip — P3-equivalent in
practice. But the class-completion is real v0.3.1 work.

## Structural debt (thermo-nuclear review, 2026-05-27)

blockers that the three correctness-review rounds missed. None are behavior
bugs — they are abstraction/size/spaghetti regressions that v0.3.0
accumulated and that will compound if the next feature lands on top of them.
**All items below are pure restructurings — no behavior change, no new
tests; the existing 427-test suite is the safety net. Imports move; logic
does not.** Do B4–B6 BEFORE the next feature touches tick.py or ledger.py.

### B6 (P0). Decompose `_tick_body_inner` (466 LOC — the single worst function in the codebase)

It inlines ~9 concerns (read+predicate, iteration check, predicate-met
short-circuit ×2, stall detection, phase-advance dispatch, seam pause,
operator guidance, rearm construction). Every fix-pass (F2's try/except,
G2's recipe-bug split) added another branch to this monolith because that
was the path of least resistance — textbook spaghetti growth.

Re-frame as a flat dispatcher over `Optional[intent]` short-circuit helpers:

```python
def _tick_body_inner(repo_root, run_id, *, adapter, auto, delay):
    led = _read_and_lock(repo_root, run_id)
    if (intent := _try_iteration_check(repo_root, run_id, led)) is not None:
        return intent
    if (intent := _try_predicate_met_shortcircuit(led)) is not None:
        return intent
    halted_ids = _detect_and_halt_stalls(repo_root, run_id, led)
    advance_result = _dispatch_phase_advance(repo_root, run_id, led, adapter, halted_ids)
    if (intent := _try_seam_pause(advance_result, repo_root, run_id, led)) is not None:
        return intent
    if (intent := _try_post_advance_predicate_met(led)) is not None:
        return intent
    return _build_rearm_intent(led, advance_result, delay)
```

Each `_try_*` is ~30-60 LOC and does one thing. F2's try/except lives in
`_try_iteration_check`; G2's recipe-bug branch lives there too. New features
go into the relevant helper, not the dispatcher. Deletes ~400 LOC of
orchestration tangle.

### B7 (P1). Decompose `recompute_predicate` (170 LOC — densest function in the codebase)

The I-1 chokepoint computes terminality + severity gating + gap counting +
iteration_pending in one body with a 60-line docstring. Extract pure
helpers:

```python
def recompute_predicate(ledger: dict) -> dict:
    severities = _count_severities_by_unit(ledger)
    terminality = _compute_terminality(ledger)        # current-phase + global
    gaps_open = _read_cached_gaps_open(ledger)         # the nullable Bug #5 field
    iteration_pending = _compute_iteration_pending(ledger)  # already extracted
    met = _evaluate_met(ledger, severities, terminality, gaps_open, iteration_pending)
    return {**severities, **terminality, "gaps_open": gaps_open,
            "iteration_pending": iteration_pending, "met": met}
```

Future iteration/bound additions land in `_compute_iteration_pending` (where
G2 + H already correctly put the shape guards) rather than growing the
chokepoint body.

### B9 (P2). Unify `exit_reason` + `bound_override` under one ledger field

Two parallel "why did the run end" envelopes: `exit_reason` (top-level,
written on iteration-check crash) and `bound_override` (per-unit
dispatch_context, written on clean bound trip). Both carry a kind + ISO
timestamp + a decision-ish payload; both render in `/auto-status` via
separate paths.

Unify under top-level `exit_reason: {kind: "predicate-met" | "bound-breach"
| "iteration-check-failed" | "recipe-bug", ...}`. Bound-breach still writes
the per-unit `bound_override` for the unit-level audit trail, but the
TOP-LEVEL `exit_reason` becomes the single "why" surface. Render goes from
two paths to one; operator looks in one place. **Contract change** —
migrate the F1 render at `lib/auto-status.py:263-269` and any
`bound_override` consumers. Needs its own mini-plan.

### B10 (P3). Move `_no_emit` to `lib/emitters.py` REGISTRY (round-2 kieran-r2-2, still open)

`_no_emit` (tick.py:559) is a 1-line Null Object emitter living in tick.py,
registered nowhere, called from one site in `advance_iteration_loop`. Either
move it to `lib/emitters.py` next to the REGISTRY (where emitters live by
convention) or inline it as `emitter=lambda l, p: []` at the call site. As

### B11 (P3). `EXIT_REASON_*` → `StrEnum`

H added three top-level names for two values
(`EXIT_REASON_ITERATION_CHECK_FAILED`, `EXIT_REASON_RECIPE_BUG`,
`EXIT_REASON_KINDS`). A `class ExitReason(StrEnum)` gives both named access
(`ExitReason.RECIPE_BUG`) AND the validation set (`list(ExitReason)`) with
type-checker support — one declaration instead of three. Compare
`iteration.DECISIONS` (plain tuple, consumers use literals) — pick ONE
convention across the codebase and apply it to both.

## Meta-finding (worth a feedback memory)

The three v0.3.0 correctness-review rounds (33 reviewer runs) had a
**structural blind spot**: none flagged "lib/tick.py crossed 1k" or
"_tick_body_inner is 466 LOC." The personas checked correctness, atomicity,
contracts, security, agent-readability — but not abstraction SHAPE or
file/function size growth. A `ce-structure-reviewer` persona (or a
mechanical size-delta lint in `tests/`) would have caught B4–B7 in round 1,
before the debt compounded across three rounds. Consider a size-budget lint:
fail the suite if any `lib/*.py` crosses 1000 LOC or any function crosses
~120 LOC without an explicit allowlist entry — the same
`feedback_deterministic_over_probabilistic_v1` mechanical-defense shape that
G5/H applied to the doc-drift class.
